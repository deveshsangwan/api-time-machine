# Domain documentation

This repository uses a single-context domain-documentation layout.

Before changing or specifying the system, read the root `CONTEXT.md` when present and any relevant decisions under `docs/adr/`. If those files do not yet exist, proceed without inventing domain rules.

Use the glossary vocabulary from `CONTEXT.md` consistently in implementation, tests, issues, and PRDs. Surface any conflict with an existing architecture decision instead of silently overriding it.
